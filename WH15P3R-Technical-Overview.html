Executive Summary

WH15P3R is a zero-storage, post-quantum encrypted communication platform supporting text, voice, and video. It provides quantum-resistant end-to-end encryption through ML-KEM (Kyber) hybrid key exchange in DTLS 1.3, with direct peer-to-peer communication and ephemeral session keys.

Core Security Features

Post-Quantum Encryption

  • Uses ML-KEM (Kyber) hybrid key exchange in DTLS 1.3
  • Protects against "harvest now, decrypt later" quantum attacks
  • SRTP (Secure Real-time Transport Protocol) for media encryption with same PQ-resistant keys
  • NIST-standardized ML-KEM (FIPS 203) quantum-resistant algorithms
Supported Browsers (PQ-Capable)
  • Chrome/Edge 142+ or Brave 1.73+ (Chromium-based)
  • Firefox 120+ or Tor Browser 14.0+ (Firefox-based)
  • Safari 17.2+ (TLS PQ support)

Note: Non-PQ browsers are blocked from connecting to enforce quantum-resistance.

Zero Metadata Architecture

  • No user accounts - No phone numbers, emails, or registration required
  • No storage - No message/call history stored anywhere (not even encrypted)
  • Ephemeral keys - Session keys destroyed on disconnect, cannot be reconstructed
  • Minimal server role - Server only facilitates initial WebRTC handshake (~1-2 seconds), then connection closes

Direct Peer-to-Peer Architecture

  • WebRTC establishes direct P2P connection between users
  • After initial handshake, all traffic bypasses servers entirely
  • No relay servers record metadata or traffic patterns
  • Low latency, high quality (depends on users' internet, not server bandwidth)

Technical Architecture

Session Establishment Flow

  1. User A selects mode (text/voice/video) and clicks "START SESSION"
  2. Browser generates ephemeral session code (cryptographically random)
  3. User A shares code with User B via separate secure channel (phone, in-person)
  4. WebRTC signaling via WebSocket to coordinate initial handshake
  5. DTLS 1.3 handshake with ML-KEM key exchange establishes PQ-encrypted channel
  6. ICE/STUN negotiation establishes direct P2P connection
  7. Signaling server disconnects - all subsequent traffic is P2P
  8. Session ends: Keys destroyed, media tracks stopped, no trace remains

Encryption Stack

Layer Protocol Algorithm
Key Exchange DTLS 1.3 ML-KEM (Kyber) hybrid
Media Encryption SRTP AES-GCM (keys from PQ handshake)
Data Channel SCTP over DTLS Same PQ keys

Deployment Information

Infrastructure Requirements

  • Static hosting: GitHub Pages (or any HTTPS static host)
  • Signaling server: Minimal WebSocket server (Deno Deploy, Node.js)
  • No database: Everything ephemeral, in-memory only
  • Bandwidth: Minimal (signaling only), P2P handles all media

Cost Analysis

  • Static hosting: $0 (GitHub Pages)
  • Signaling server: $0-5/month (Deno Deploy free tier)
  • Media bandwidth: $0 (P2P means users pay their own ISP)
  • Total: ~$0/month for thousands of concurrent sessions

Security Recommendations

For Maximum Security:
  1. Use Tor Browser - Hides IP address, prevents traffic correlation
  2. Verify session codes out-of-band - Phone call or in-person to prevent MITM
  3. Use from public WiFi - Random locations, never home/work
  4. Text-only mode preferred - Avoids camera/mic permissions
  5. Close immediately after use - Minimize exposure window
  6. Never screenshot or record - Defeats ephemeral nature

Use Cases

Ideal For:

  • Journalists communicating with sources
  • Whistleblowers sharing sensitive information
  • Activists in authoritarian countries
  • High-risk temporary communications
  • Anyone requiring zero-trace communications

Not Ideal For:

  • Casual daily messaging (use Signal)
  • Persistent contact management
  • File sharing (no storage mechanism)
  • Group conversations (currently 1-on-1 only)

Conclusion

WH15P3R fills a unique niche in the secure communications landscape: maximum-security temporary communications where identity anonymity and zero-trace requirements outweigh convenience. Unlike Signal (persistent contacts, phone numbers) or traditional messengers, WH15P3R provides:

  • True post-quantum encryption for all communication modes
  • Zero metadata generation and storage
  • No user identity or registration
  • Ephemeral sessions with no trace after disconnect

It represents the current state-of-the-art in quantum-resistant, privacy-preserving communications technology.
